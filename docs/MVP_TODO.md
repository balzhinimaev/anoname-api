### 1) Блокеры (CRIT) — до запуска MVP

- Загрузка рейтинга пишется в неверное поле (ломает консистентность профиля)
  - Файл(ы): `src/services/RatingService.ts`
  - Проблема: запись в `stats.rating`, в модели `User` такого поля нет. Должно быть в `user.rating`.
  - Влияние: рейтинг пользователя не обновляется/не виден клиенту.
  - Действия: заменить путь на корректный, пересчитать и обновить средний рейтинг после оценки.
  - Критерий приемки: после события `chat:rate` у оценённого пользователя поле `rating` обновляется, значение стабильно между перезапусками.
  - Приоритет/Статус/Оценка: CRIT / TODO / S

- REST ChatController пишет в несуществующие поля (`messages` в `Chat`)
  - Файл(ы): `src/controllers/chatController.ts`, модель `src/models/Chat.ts`
  - Проблема: контроллер пушит сообщения в поле `messages`, которого нет в схеме. Сообщения — отдельная коллекция `Message`.
  - Влияние: REST-эндпоинт отправки сообщений работает некорректно; данные не соответствуют модели.
  - Действия: либо удалить/скрыть неиспользуемые REST-методы сообщений, либо привести к работе через `Message` (как в `ChatService`).
  - Критерий приемки: REST-эндпоинты чатов либо корректны и согласованы со схемой, либо скрыты из Swagger/недоступны.
  - Приоритет/Статус/Оценка: CRIT / TODO / M

- Обязательность секретов JWT, единый источник
  - Файл(ы): `src/config/index.ts`, `src/middleware/authMiddleware.ts`
  - Проблема: дефолт `your-secret-key`. Разные места берут секрет по-разному.
  - Влияние: риск компрометации, несогласованность проверки токенов.
  - Действия: убрать дефолт, требовать `JWT_SECRET` из окружения, везде использовать `config.jwtSecret`.
  - Критерий приемки: при отсутствии `JWT_SECRET` процесс не стартует; токены в REST/WS валидируются одинаково.
  - Приоритет/Статус/Оценка: CRIT / TODO / S

- Поиск потенциальных партнёров: фильтр по несуществующему полю
  - Файл(ы): `src/controllers/userController.ts`
  - Проблема: в запросе стоит `isActive: true` у `User`, такого поля нет. Вероятно подразумевается `isOnline` или убрать вовсе.
  - Влияние: пустые/некорректные ответы.
  - Действия: заменить/убрать фильтр.
  - Критерий приемки: эндпоинт возвращает ожидаемые данные.
  - Приоритет/Статус/Оценка: CRIT / TODO / S

### 5) Чек-лист конфигурации и деплоя (обязателен для запуска)

- Env: `JWT_SECRET` (без дефолта), `MONGODB_URI`, `PORT=3001`, `CLIENT_URL=https://<your-domain>`
- YooKassa: `YOOKASSA_MODE`, `*_SHOP_ID_*`, `*_SECRET_KEY_*`, webhook basic-auth `YOOKASSA_WEBHOOK_USER/PASSWORD`
- Nginx: отдельный `location /socket.io/` с Upgrade/Connection; `location /api/`; фронт — по `/`
- TLS: валидные серты, автообновление (Certbot на проде присутствует)
- Логи: права на каталог `logs/`, ротация файлов, отсутствие секретов в логах

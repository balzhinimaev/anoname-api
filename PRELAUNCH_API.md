# Prelaunch API Documentation

## GET /api/prelaunch/stats

Получает расширенную статистику прелаунча с информацией о текущем пользователе.

### Аутентификация
Требуется Bearer токен в заголовке `Authorization`.

```
Authorization: Bearer <your-jwt-token>
```

### Ответ

#### Успешный ответ (200 OK)
```json
{
  "totalCount": 9,
  "userPosition": 5,
  "isInQueue": true,
  "joinedAt": "2024-01-15T10:30:00.000Z"
}
```

#### Поля ответа:
- `totalCount` (number) - общее количество пользователей в очереди прелаунча
- `userPosition` (number) - позиция текущего пользователя в очереди (1 = первый в очереди)
- `isInQueue` (boolean) - находится ли пользователь в очереди прелаунча
- `joinedAt` (string, optional) - дата и время присоединения к очереди (ISO 8601, только если `isInQueue: true`)

#### Ошибки:
- `401 Unauthorized` - отсутствует или недействительный токен
- `500 Internal Server Error` - ошибка сервера

### Примеры использования

#### Пользователь в очереди:
```bash
curl -H "Authorization: Bearer <token>" \
     http://localhost:3000/api/prelaunch/stats
```

Ответ:
```json
{
  "totalCount": 15,
  "userPosition": 3,
  "isInQueue": true,
  "joinedAt": "2024-01-15T10:30:00.000Z"
}
```

#### Пользователь не в очереди:
```json
{
  "totalCount": 15,
  "userPosition": 0,
  "isInQueue": false
}
```

### Логика определения позиции

Позиция пользователя определяется по времени присоединения к очереди (`joinedAt`):
- Пользователи, присоединившиеся раньше, имеют меньший номер позиции
- Позиция 1 означает, что пользователь первый в очереди
- Если пользователь не в очереди, `userPosition` = 0

### Безопасность

- Эндпоинт защищен JWT аутентификацией
- Токен должен быть действительным и не истекшим
- Пользователь может видеть только свою позицию в очереди
